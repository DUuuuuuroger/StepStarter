# StepStarter

StepStarter 是一款桌面应用，帮助用户在面对目标或任务时迅速找到「第一步」，降低启动成本，避免拖延与复杂性带来的退缩。项目采用本地运行的架构，使用 PyInstaller 打包为 `exe`，NSIS 打包为 `exe` 安装包，以便更易于分发和部署。

## 1. 项目背景（Problem Statement）

**核心痛点**
- 面对目标或复杂任务时，用户常卡在「不知道从哪里开始」。
- 任务拆解成本高，容易拖延，最终导致目标搁置。

**现实场景**
- 学习：想系统学习一门技能，但不知道第一步要做什么。
- 工作：接到复杂项目，初期推进困难，迟迟无法启动。
- 生活：想改善习惯或完成个人计划，却不断推迟。

**不解决的后果**
- 目标停留在想法层面，缺少落地路径。
- 启动门槛过高导致长期拖延与自我否定。
- 任务复杂度被高估，效率与信心持续下降。

## 2. 解决方案（Solution）

**StepStarter 的做法**
- 以「可执行的第一步」为核心，把目标压缩成可启动的最小行动。
- 通过 AI 步骤生成与建议机制，引导用户快速进入行动状态。
- 本地化存储任务与进度，保持连续性与可回溯性。

**解决到的程度**
- 提供从「目标描述」到「步骤列表」的完整转化。
- 让用户在数分钟内完成可执行计划的起步。

**相较传统方法的优势**
- 传统方法依赖用户自我拆解，成本高且不稳定。
- StepStarter 用「最小行动」策略降低启动门槛，并用系统化流程保持持续推进。

## 3. 对学习 / 工作 / 生活的影响（Impact）

**个人效率**
- 减少启动延迟，提升完成率。
- 用清晰的第一步降低认知负担。

**长期成长**
- 形成可复用的任务拆解思路。
- 更容易累积长期项目的推进记录。

**团队 / 职场价值**
- 任务更可拆解、可跟踪，便于协作与复盘。
- 有助于提升执行节奏与项目可控性。

## 4. 技术架构（Tech Stack）

**前端**
- HTML / CSS / JavaScript（本地页面）
- 通过 PyWebView 加载并运行 UI

**后端**
- Python + FastAPI（本地 API 服务）
- Uvicorn 作为本地服务运行器

**数据库**
- SQLite（本地轻量存储，零配置）

**打包方式**
- PyInstaller（打包为 Windows `exe`）
- NSIS（打包为安装包）

**选择理由**
- PyWebView 兼具桌面体验与 Web UI 的开发效率。
- FastAPI 结构清晰，便于扩展 API 与服务模块。
- SQLite 简洁可靠，适合本地个人数据存储。
- PyInstaller + NSIS 易于分发与部署，降低安装门槛。

## 5. 项目亮点（Highlights）

- **AI 步骤生成**：根据目标自动生成可执行步骤（可配置 `base_url`、`api_key` 与模型）。
- **本地建议机制**：基于当前任务状态，给出下一步启动建议。
- **本地部署优势**：核心功能无需联网即可运行，数据不离开本地。
- **体验优化**：启动即用，无需复杂配置；支持流式输出步骤结果。
- **架构清晰**：前端 UI、后端服务与数据层解耦，便于扩展与维护。

## 6. 使用方式（How to Use）

**使用（推荐）**
- 从 Releases 下载对应的安装包并完成安装。
- 启动应用后，在设置中填入 `API` 配置即可使用。
